# ⚔️ 勇者挂机传说

一款基于原生 HTML/CSS/JavaScript 的放置类挂机 RPG 游戏。无需安装，直接在浏览器中打开 `index.html` 即可游玩。支持浏览器后台运行计算，长时间挂机也不会丢失进度。

---

## 🎮 游戏简介

你扮演一名勇者，通过不断击败怪物来成长变强。探索 **15 个**风格迥异的区域，挑战 **4 个**强大的区域 BOSS，最终解锁 **无尽模式** 挑战无限层数。沿途收集稀有宝物、装备和高阶技能书，学习强大的魔法技能并给敌人施加负面状态，收集被动技能书获得元素抗性以抵抗敌人的 debuff，打造属于你的最强勇者！

---

## 🚀 快速开始

1. 直接用浏览器打开 `index.html`（或 `dist/index.html` 单文件版）
2. 点击 **"开始挂机"** 按钮即可自动战斗
3. 点击 **"🏰 主城"** 前往 NPC 处学习技能、锻造装备、购买道具、休息恢复
4. 点击 **"手动攻击"** 可立即发动一次攻击

## 📋 核心玩法

### 战斗系统

| 操作 | 说明 |
|------|------|
| 开始挂机 | 开启自动战斗，勇者会自动攻击敌人并释放技能 |
| 手动攻击 | 立即发动一次攻击 |
| 自动恢复 | 战斗中自动使用药水恢复生命 / 魔力 |
| 释放技能 | 点击技能按钮消耗魔力释放技能（有冷却时间） |

**伤害公式（v2 新公式）**：

- 基础伤害 = `max(1, 攻击力² / (攻击力 + 敌方防御))`
- 暴击时伤害 × 暴击伤害倍率
- 41 级以上攻速段额外乘以伤害倍率（最高 ×3）
- 破甲（固定值）：无视敌人 `破甲值 × 1.5` 的防御
- 破甲（百分比）：无视敌人 `百分比` 的防御（上限 35%）

> 新公式使得"攻防比"成为关键指标：攻击力远高于防御时接近全额命中，攻防接近时输出衰减明显，鼓励持续培养攻击力。

**死亡惩罚**：

- 普通战斗：损失 `min(500, 金币 × 5% + 50)`，30% 概率丢失一件未锁定的宝物
- BOSS 战：损失相同金额的金币，BOSS 逃走并清空线索

| # | BOSS | 所需线索 | 专属技能 | 克制手段 | 击败奖励 |
|---|------|---------|---------|---------|---------|

| 10 | ☁️ 天空领主 | 20 | 雷霆审判（10% 生命雷伤）+ 眩晕打击 | 雷抗减少雷伤；冰技能重置雷霆 CD | 同上 |

**线索机制**：

- 进入 BOSS 区域后，击败普通怪有 **15%** 概率掉落线索
- 收集足够线索后点击 **"前往 BOSS 房间"** 即可挑战
- 击败 BOSS 后该区域永久解锁，无需再次挑战
- 被 BOSS 击败后，BOSS 逃走、线索清零

### ⚡ 精英怪物

- BOSS 区域 **30%** 概率生成精英；其它区域 **15%** 概率
- HP / 攻 / 防 显著高于普通怪，并随机持有 1~2 个基础技能
- 掉落倍率 ×1.5，宝物/装备最低**稀有**品质
- **10%** 概率掉落**精英核心**（永久 +2 攻击力）
- **30%** 概率掉落**强化石**

### ♾️ 无尽模式

- 解锁条件：击败 **混沌魔神**（区域 15）
- 主城地图末尾出现 **"♾️ 无尽模式"** 入口
- 击败一只敌人即进入下一层
- 怪物等级 = `100 + 2 × 层数`，难度倍率 = `1.08^层数`
- **100 级角色正常可打到 20 层以上**
- 每 5 层必出精英，每 10 层必出 BOSS
- 普通小怪 **1% 概率掉落神器装备/宝物**，随层数提升掉落恢复/加成道具（最高 15%）
- 宝物基础掉率 **25%**，稀有度权重随层数动态提升（层数越高，史诗/传说/神器概率越大）
- 精英/Boss 神器装备掉率最高 **25%**，超脱套装/无尽道具/禁咒书掉率最高 **20%**
- 适合长期挂机刷稀有装备、超脱级套装与禁咒技能书

## 🔮 技能系统

### 精神与魔力

| 属性 | 说明 | 获取方式 |
|------|------|---------|
| 🧠 精神 | 影响技能伤害倍率和魔力上限 | 升级 +1、精神修炼 +2 |
| 💧 魔力 | 释放技能消耗的资源 | 普攻 +1，战斗外按精神回复 |

**魔力规则**：

- 最大魔力 = `50 + 精神 × 3 + 等级 × 2`
- **战斗外恢复**：战斗结束 30 秒后开始，每秒回 `max(1, floor(精神 × 0.5))` 魔力
- 每次普通攻击恢复 1 点魔力
- 升级时魔力回满

## 🏰 主城 NPC

点击顶部 **"🏰 主城"** 按钮进入主城，与 5 位 NPC 交互：

### 🧙‍♂️ 技能导师

- **学习技能**：花费金币学习基础技能
- **升级技能**：花费金币提升已学会的技能等级
- **精神修炼**：花费金币提升精神力（+2 精神，+6 魔力上限）

### 👨‍🎓 鉴定师

- **鉴定装备**：鉴定后才能穿戴
- **鉴定技能书**：揭示技能书中包含的技能
- 鉴定费用 = 物品售价的 50%~80%
- 列表支持按 **装备 / 技能书** 切换筛选，并按品质从高到低排序
- 子视图顶部筛选栏固定，下方列表独立滚动

### 👷‍♂️ 铁匠

- **打造装备**：花费金币随机打造一件装备（品质随等级提升）
  - **无需鉴定**：打造装备直接可用，省去鉴定环节
  - **属性浮动**：非神器装备基础属性在 **50%~120%** 之间随机浮动，显示数值即为浮动后的最终属性
  - **神器机率**：**万分之一** 概率打造神器，属性固定且与无尽模式掉落保持一致（无浮动）
- **锻造强化**：花费金币强化装备，提升属性
- 锻造成功率随锻造等级下降，**强化石**可提升成功率（每个 +5%，最高 100%）
- 锻造失败：锻造等级 -1（+0 时装备损毁）
- 装备列表按品质从高到低排序，支持按部位筛选

## 🎒 宝物系统

从第 4 关起，击败怪物有几率掉落宝物。宝物分为 5 个稀有度：

| 稀有度 | 颜色 | 权重 | 说明 |
|--------|------|------|------|
| 普通 | 灰色 | 55% | 基础属性加成 |
| 稀有 | 蓝色 | 30% | 中等属性加成 |
| 史诗 | 紫色 | 12% | 强力属性加成 |
| 传说 | 橙色 | 3% | 顶级属性加成 |
| 神器 | 红色 | 1% | 无尽模式专属 |

### 宝物属性类型

- **攻击** / **防御** / **生命**：基础三维属性
- **暴击伤害**：提升输出上限
- **经验加成** / **金币加成**：提升资源获取效率
- **破甲（固定值）**：无视敌人固定数值的防御
- **破甲（百分比）**：无视敌人百分比防御（上限 35%）
- **元素抗性**：土抗/毒抗/雷抗/虚空抗/混沌抗/火抗（上限 40%，克制 BOSS 技能）

### 宝物操作

| 功能 | 说明 |
|------|------|
| 强化 | 消耗 1 个同名宝物 + 金币，提升宝物等级（属性 +50%/级） |
| 出售 | 获得金币，锁定的宝物无法出售 |
| 锁定 | 点击 🔓 锁定宝物，防止误操作出售或死亡掉落 |
| 筛选 | 可按属性类型筛选显示宝物 |
| 自动强化 | 自动消耗金币和多余宝物进行强化（需开启开关） |
| 排序 | 背包内宝物按品质从高到低自动排序 |

## 🛠️ 技术说明

### Docker 部署（推荐生产 / 自托管）

一键启动整套服务（前端 + 后端 + Redis 缓存 + MySQL 持久化）：

```bash
# 1. 复制环境变量模板并修改（生产环境务必修改密码和盐值）
cp .env.example .env
# vim .env

# 2. 启动服务
docker compose up -d
# 浏览器访问 http://localhost:8080
```

涉及文件：
- `Dockerfile`：基于 `php:8.2-apache`，预装 `pdo_mysql / redis / opcache` 扩展，含健康检查
- `docker-compose.yml`：编排 `web / redis / mysql` 三个服务，含数据卷与健康依赖
- `.env`：环境变量配置（被 `.gitignore` 排除，需从 `.env.example` 复制）
- `.dockerignore`：排除 `.git / server/data / 本地配置` 等

环境变量（在 `.env` 中调整，`docker-compose.yml` 引用）：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `APP_ENV` | `production` | 环境标识 |
| `APP_DEBUG` | `false` | 错误堆栈输出 |
| `AUTH_DEVICE_SALT` | `change-this-in-production` | **务必修改**，设备 ID 哈希盐 |
| `REDIS_ENABLED` / `REDIS_HOST` / `REDIS_PORT` | `true / redis / 6379` | Redis 缓存层 |
| `MYSQL_ENABLED` / `MYSQL_HOST` / `MYSQL_DATABASE` / `MYSQL_USERNAME` / `MYSQL_PASSWORD` | `true / mysql / iqotb / iqotb / iqotb_pwd_change_me` | MySQL 持久层 |
| `MYSQL_ROOT_PASSWORD` | `root_pwd_change_me` | MySQL root 密码 |

只跑独立 web（不要 Redis / MySQL）：

```bash
docker build -t idle-quest-of-the-brave .
docker run -p 8080:80 \
  -e REDIS_ENABLED=false -e MYSQL_ENABLED=false \
  -v iqotb-data:/var/www/html/server/data \
  idle-quest-of-the-brave
```

数据持久化：`server/data` 通过 Docker volume 挂出来，重建容器不丢存档。

健康检查：`GET /game/idle-quest-of-the-brave/health` 返回 `{"ok": true}`。

### 浏览器后台运行优化

当页面切换到后台时，浏览器的 `setInterval` 会被节流到约 1000ms。游戏通过记录时间戳，在切回前台时批量计算错过的攻击次数和技能释放，最多批量处理 500 次攻击，确保长时间挂机不会丢失进度。

### 自动恢复

开启自动恢复后，战斗中会自动使用背包中的药水：

- 生命值低于 30% 时自动使用生命药水（优先大药水）
- 魔力值低于 30% 时自动使用魔力药水
- 药水用尽后自动关闭自动恢复

## 📝 更新日志

### 🆕 v4.0 最新（联网架构重构）

> 这是一次"前端单机 → 服务端权威"的重大升级。游戏可以离线游玩，但联网时所有战斗/存档/NPC/市场都由服务端计算并持久化。

- **☁️ 服务端权威架构**
  - 原生 PHP 后端（`server/`）：账号 / 存档 / 战斗 / NPC / 技能 / 市场 / 进度
  - 联网模式下战斗（伤害、暴击、闪避、debuff、BOSS 技能、区域被动、掉落、经验、金币、boss 击杀、无尽推进）100% 由服务端 `BattleEngine` 计算
  - 前端只做日志渲染 + 动画 + 决策上的轻量交互（手动施法按钮等）
- **🗄️ 三级存档存储（cache-aside）**
  - **Redis**（可选启用）：读取缓存，命中直接返回
  - **MySQL**（可选启用）：仅持久化"存档信息"（与功能拆分一致：market / auth 不走 MySQL）
  - **File**（始终启用）：本地权威 + 互斥锁（`flock(LOCK_EX)`）
  - 任一层异常自动降级，存档不丢；自动建表（`CREATE TABLE IF NOT EXISTS`）
  - 配置：环境变量 / `.env` 文件（`server/.env` 或根目录 `.env`）> `config/local.php` > `config/app.php` 默认值
- **⚔️ 批量战斗回放（30 回合 / 节奏播放）**
  - 一次 `/battle/tick` 服务端固定计算 30 回合，每回合带 `ts_offset_ms / player_hp / player_mp / enemy_hp / skills / player_state` 等
  - 前端按 `aspd_ms` 节奏逐回合渲染日志 + 同步状态，营造"连续战斗"动画
  - 服务端节流：`session.next_batch_at_ms = now + duration_ms`，未播完拒绝新请求（`batch_in_progress + wait_ms`）
  - 服务器压力从"挂机时每 300ms 一次"降到"每 aspd × 30 ≈ 数十秒一次"
- **🧠 自动施法决策搬到后端**
  - 服务端 `SkillEngine::pickAutoSkill` 复刻原前端 `tryAutoCastSkill` 的优先级（低血治疗 > 没护盾上 buff > 选最强输出）
  - 批量内每回合都尝试自动施放，cd / mp 同步流逝
- **🔁 跨设备恢复存档**
  - URL `?device_hash=xxx` → 自动调用 `/auth/by-hash` 换 token 加载对应存档
  - URL 参数加载后自动清除，避免重复触发
  - 适合本地测试 / 跨设备分享存档
- **🏪 市场上线（联网正式启用）**
  - 装备 / 宝物 / 道具 / 技能书 4 类挂售，单用户最大 30 单，挂售 TTL 7 天
  - 列表区分"我的挂售"与他人挂售：自己的卡片金边 + "我的"徽章 + "取消挂售"按钮（不能买自己的）
  - `MarketController::listings` 返回每条 listing 的 `is_mine` 标记
- **💾 存档同步与一致性**
  - 联网模式下移除前端"30s 整存档 PUT"（避免覆盖服务端权威）
  - 纯前端操作（穿装备 / 出售 / 升级能力 / 锁定宝物 / 自动开关）通过 `/save/diff` 局部增量同步，刷新后不丢失
  - PUT /save 防御性兜底：未带 `battle` 字段时保留服务端原值，前端误传不会清掉战斗会话
  - PHP `json_decode` 空对象坑修复：服务端 `SaveRepository::normalize` 把空 `[]` 强制转 `stdClass {}`，前端 `_normalizePlayerObjectFields` 双重保险
- **🩹 自动恢复 / 自动出售 / 自动强化开关持久化**
  - `world.autoRecover / autoStrengthen / autoSell` 状态通过 `/save/diff` 实时同步

#### 部署提示

**Docker 部署**（推荐）：

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑 .env 修改密码和盐值

# 2. 启动
docker compose up -d
```

**本地 PHP 开发**：

```bash
# 启动 PHP 内置服务器
php -S 127.0.0.1:8787 -t server/public

# 可选：启用 Redis + MySQL（二选一）
# 方式 A：通过 .env 文件
cp server/.env.example server/.env
# 编辑 server/.env 填入 host/port/credentials

# 方式 B：通过 local.php（适合复杂配置覆盖）
cp server/config/local.example.php server/config/local.php
# 编辑 local.php 覆盖任意配置项
```

> 配置优先级：系统环境变量 > 根目录 `.env` > `server/.env` > `config/local.php` > `config/app.php` 默认值
